Help & User Guide

Complete guide to using the Federated Learning CKD System

📖 About the System

The Federated Learning Chronic Kidney Disease (FL-CKD) System is an advanced healthcare platform that enables multiple medical institutions to collaboratively train machine learning models for CKD prediction without sharing sensitive patient data.

This system implements Federated Learning with Differential Privacy to ensure data security and patient confidentiality while improving diagnostic accuracy through collaborative learning.

Key Benefit: Healthcare providers can benefit from collective intelligence without compromising patient privacy or violating data protection regulations.

✨ Key Features

🔒 Privacy-Preserving

Patient data never leaves the local institution. Only model updates are shared.

🤝 Collaborative Learning

Multiple clients train together to create a more robust global model.

🎯 Accurate Predictions

Advanced neural network models for CKD risk assessment and diagnosis.

📊 Real-time Dashboard

Monitor training progress, metrics, and model performance in real-time.

🔐 Differential Privacy

Additional privacy layer with noise injection to protect individual records.

📈 Performance Metrics

Comprehensive evaluation with accuracy, precision, recall, and F1-score.

🚀 Getting Started

1 Choose Your Portal

The system has two main portals:

  • Admin Portal: For system administrators to manage clients, monitor global training, and oversee the federated learning process.
  • Client Portal: For healthcare institutions to upload data, train local models, and make predictions.

2 Login

Use your credentials provided by the system administrator to access your respective portal.

Note: If you don't have credentials, contact your system administrator for account setup.

3 Navigate the Dashboard

Once logged in, you'll see a dashboard with multiple sections tailored to your role.

👨‍💼 Admin Portal Guide

Client Management

Administrators can:

  • Register new healthcare institutions as clients
  • View all registered clients and their status
  • Monitor client participation in training rounds
  • Manage client credentials and permissions

Global Model Management

Control the federated learning process:

  • Initialize Global Model: Set up the initial model architecture and parameters
  • Aggregate Updates: Combine model updates from all participating clients
  • Monitor Performance: Track global model accuracy and convergence
  • Distribute Model: Share the updated global model with all clients

System Monitoring

  • View training round history and statistics
  • Monitor system health and performance metrics
  • Track client participation and contribution
  • Review aggregation logs and model versions

🏥 Client Portal Guide

Data Upload & Training

Healthcare institutions can participate in federated learning:

  1. Prepare Your Dataset: Format patient data according to the required schema (CSV format)
  2. Upload Data: Securely upload your dataset through the training section
  3. Start Training: Initiate local model training on your data
  4. Submit Updates: Send encrypted model updates to the central server
Important: Your raw patient data never leaves your local system. Only encrypted model parameters are transmitted to the server.

Making Predictions

Use the trained model to predict CKD risk for new patients:

  1. Navigate to the Prediction section
  2. Enter patient information (age, blood pressure, glucose levels, etc.)
  3. Click Predict to get the CKD risk assessment
  4. Review the prediction results and confidence scores

Dashboard & Metrics

Monitor your local model performance:

  • View training history and loss curves
  • Check model accuracy, precision, and recall
  • Compare local vs. global model performance
  • Track participation in federated rounds

📋 Data Format Requirements

Required Patient Features

The CKD prediction model requires the following patient information:

  • Age: Patient age in years
  • Blood Pressure: Systolic/Diastolic measurements
  • Specific Gravity: Urine specific gravity
  • Albumin: Albumin levels in urine
  • Sugar: Sugar levels in urine
  • Red Blood Cells: RBC count
  • Pus Cells: Presence of pus cells
  • Blood Glucose: Random blood glucose levels
  • Blood Urea: Blood urea nitrogen levels
  • Serum Creatinine: Creatinine levels in blood
  • Sodium/Potassium: Electrolyte levels
  • Hemoglobin: Hemoglobin count
  • And other relevant clinical parameters

CSV File Format

Upload data in CSV format with proper column headers. Example:

age,bp,sg,al,su,rbc,pc,bgr,bu,sc,sod,pot,hemo,classification
48,80,1.020,1,0,normal,abnormal,121,36,1.2,138,4.5,15.4,ckd
53,90,1.020,2,0,abnormal,normal,70,107,7.2,114,3.7,9.4,ckd

🔐 Privacy & Security

How Your Data is Protected

  • Local Training: All patient data remains on your local server
  • Encrypted Communication: Model updates are encrypted during transmission
  • Differential Privacy: Noise is added to model updates to prevent data inference
  • Secure Aggregation: Server cannot access individual client data
  • Access Control: Role-based authentication and authorization

Compliance

The system is designed to comply with healthcare data protection regulations including HIPAA, GDPR, and other regional privacy laws. Patient data never leaves the healthcare institution's premises.

🔧 Troubleshooting

Common Issues

Cannot Login

  • Verify your username and password are correct
  • Check if your account is active
  • Contact the administrator if you've forgotten your credentials

Upload Fails

  • Ensure your CSV file is properly formatted
  • Check that all required columns are present
  • Verify file size is within limits
  • Make sure there are no special characters in column names

Training Errors

  • Verify your dataset has sufficient samples (minimum 100 records recommended)
  • Check for missing values in critical columns
  • Ensure data types are correct (numeric values for measurements)
  • Review error messages in the dashboard for specific issues

Prediction Not Working

  • Ensure a model has been trained first
  • Verify all required patient information is provided
  • Check that input values are within valid ranges
  • Try refreshing the page and logging in again
